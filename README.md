## How to Install the Application

1. Download or clone the project.
2. Open the project folder in a terminal.
3. Install dependencies:

```bash
npm install
```

## Supabase Setup

This project uses one simple Supabase table called `favorites`.

Create this table in Supabase:

```sql
create table favorites (
  id bigint generated by default as identity primary key,
  team_name text not null,
  team_abbrev text not null,
  note text,
  created_at timestamp with time zone default timezone('utc'::text, now())
);
```

Create a `.env` file in the main project folder:

```env
SUPABASE_URL=your_supabase_project_url
SUPABASE_ANON_KEY=your_supabase_anon_key
```

If Supabase is not configured, the app still runs with local fallback demo data. However, for the final rubric, Supabase should be connected.

## How to Run the Application on a Server

Run the server locally:

```bash
npm start
```

or:

```bash
node server.js
```

Then open this URL in a browser:

```txt
http://localhost:3000
```

## How to Run Tests

There are no automated tests written for this intro project yet.

Run the test script:

```bash
npm test
```

Manual testing steps:

1. Open the home page and check that upcoming games load.
2. Open the dashboard page and check that scores and the Chart.js chart load.
3. Open the favorites page and add a favorite team.
4. Refresh the favorites page and check that the saved team appears.
5. Test backend endpoints directly in the browser or with Insomnia.

## Server API Endpoints

### GET `/api/health`

Checks if the backend server is running.

Example response:

```json
{
  "ok": true,
  "message": "NHL Insight Hub server is running."
}
```

### GET `/api/scores`

Gets current NHL score data from the NHL API.

Used by: `dashboard.html`

Example response:

```json
{
  "source": "NHL API",
  "games": [
    {
      "id": 1,
      "away": "Rangers",
      "home": "Capitals",
      "awayScore": 3,
      "homeScore": 4,
      "state": "Final",
      "gameTime": "May 17, 7:00 PM"
    }
  ]
}
```

### GET `/api/schedule`

Gets upcoming NHL schedule data from the NHL API.

Used by: `index.html`

### GET `/api/leaders`

Gets NHL standings from the NHL API and displays the top teams by points.

Used by: `dashboard.html`

### GET `/api/favorites`

Reads favorite teams from the Supabase database.

Used by: `favorites.html`

### POST `/api/favorites`

Writes a new favorite team to the Supabase database.

Used by: `favorites.html`

Required request body:

```json
{
  "team_name": "Washington Capitals",
  "team_abbrev": "WSH",
  "note": "My favorite team"
}
```

## External Data Sources

### NHL API

Base URL:

```txt
https://api-web.nhle.com/v1
```

Endpoints used by the backend:

```txt
GET /score/now
GET /schedule/now
GET /standings/now
```

### Supabase

Supabase stores user-submitted favorite teams in the `favorites` table.

## Frontend Libraries

This project uses two frontend libraries:

1. Chart.js - creates the standings bar chart.
2. AOS - adds simple scroll/fade animations to cards.

Supabase is used for the backend/database requirement, but it is not counted as a frontend library.

## Known Bugs

- If the NHL API is down or there are no current games, the app displays demo fallback data.
- If Supabase environment variables are missing, the favorites page uses local fallback data instead of the real database.
- The app does not currently support deleting favorite teams.
- The app does not currently allow searching for a specific team.

## Deployment Notes for Vercel

The project includes `vercel.json` so Vercel can use `server.js` as the backend entry file.

Before deploying, add these environment variables in the Vercel project settings:

```txt

SUPABASE_ANON_KEY
```
